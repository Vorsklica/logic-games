# Журнал розробки

## Гра Slide

**Дата:** 23.07.2026

### Прийняті архітектурні рішення

Під час проєктування нової гри було прийнято рішення створити універсальну реалізацію класичного ковзного пазла.

---

## Назва гри

- Робоча назва (ідентифікатор): **slide**
- Назва, що відображається користувачу, визначається набором даних (наприклад: «Пазл 3×3», «Пазл 4×4», «Пазл 5×5»).

---

## Загальна концепція

Гравець складає ковзний пазл із пронумерованих плиток.

Кожна плитка містить:

- порядковий номер;
- відповідний фрагмент зображення.

Числа допомагають розв'язувати головоломку, але не заважають сприйняттю картинки.

---

## Завершення гри

Після встановлення останньої плитки:

1. зупиняється таймер;
2. фіксується кількість виконаних ходів;
3. цифри на плитках плавно зникають;
4. зібране зображення відображається без номерів;
5. користувач бачить результат проходження;
6. відображається кнопка **«Зіграти ще раз»**.

При повторі використовується той самий набір даних, але виконується нове випадкове перемішування плиток.

---

## Статистика

Під час гри ведеться:

- таймер;
- лічильник ходів.

Збереження рекордів на першому етапі не передбачається.

---

## Контент

Кожен набір (`data-xxx`) містить:

- одне зображення;
- файл `info.js`, який описує параметри набору.

Розмір поля є властивістю набору даних і зберігається в `info.js`.

Для запуску гри достатньо одного параметра URL:

```text
?set=7
```

## Структура info.js

```javascript
export default {
  id: "data-001",
  title: "Пазл 4×4",
  size: 4,
};
```

Назва файлу зображення стандартизована:

```text
image.jpg
```

Тому її не потрібно дублювати в `info.js`.

---

## Архітектурні принципи

- Один набір (`data-xxx`) — один пазл.
- Один параметр URL — `set`.
- Уся інформація про набір зберігається в `info.js`.
- Розмір поля визначається властивістю `size`.
- Ігрова логіка працює з номерами плиток, а зображення використовується лише для візуального оформлення.
- Алгоритм перемішування повинен завжди створювати розв'язувану конфігурацію.
- Таймер запускається після першого ходу гравця.

---

## Подальші етапи

1. Створити файлову структуру гри.
2. Реалізувати відображення ігрового поля.
3. Реалізувати алгоритм перемішування.
4. Реалізувати переміщення плиток.
5. Реалізувати перевірку завершення гри.
6. Додати анімацію завершення.
7. Додати таймер і лічильник ходів.
8. Реалізувати кнопку «Зіграти ще раз».
