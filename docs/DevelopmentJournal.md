### Отримані знання

Вивчено основні принципи роботи Telegram Bot API.

З'ясовано, що:

- бот є окремим акаунтом Telegram;
- бот не працює всередині Telegram — ним керує зовнішня програма;
- Telegram виступає посередником між користувачем і програмою;
- взаємодія відбувається через HTTPS-запити до Telegram Bot API;
- бот і канал є незалежними об'єктами, які потрібно окремо пов'язати.

2026-07-16 — Проєктування механізму публікації

Сьогодні робота була присвячена не програмуванню, а проєктуванню архітектури автоматичної публікації ігор у Telegram.

У ході обговорення було прийнято такі рішення:

відмовитися від створення окремих JSON-файлів для кожної публікації (post-001.json, post-002.json тощо);
використовувати один робочий файл bot/post.json, який містить опис поточного поста;
відмовитися від ведення окремої історії публікацій (history, posts.csv тощо), оскільки для розважального каналу вона не має практичної цінності;
визначити базову модель поста Telegram: необов'язкове медіа → текст → кнопки;
погодитися, що формат post.json повинен відображати реальні можливості Telegram і не підтримувати конструкції, яких Telegram не вміє;
прийняти структуру опису гри через об'єкт game з обов'язковим полем id та необов'язковим полем set;
домовитися, що якщо гра використовує набори завдань, вони завжди розташовуються у каталозі content/ всередині папки гри;
вирішити не ускладнювати формат даних заради універсальності та використовувати єдине поле set для всіх ігор, які працюють із наборами даних.

У результаті архітектура механізму публікації стала простішою, зрозумілішою та легшою для подальшого розвитку.
