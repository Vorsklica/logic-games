# Робота з Telegram Bot API

## Що таке Telegram-бот

Telegram-бот — це спеціальний обліковий запис Telegram, яким керує програма.

Бот не працює всередині Telegram.

Він запускається на комп'ютері або сервері, а Telegram лише передає між користувачем і програмою повідомлення.

Схема роботи:

Користувач
↓
Telegram
↓
Telegram Bot API
↓
Наша програма

## BotFather

BotFather — офіційний сервіс Telegram для створення та налаштування ботів.

Основні можливості:

- створення нового бота;
- зміна назви;
- зміна опису;
- встановлення фотографії;
- налаштування команд;
- отримання API Token.

## API Token

API Token — секретний ключ, який дозволяє програмі керувати ботом.

Токен необхідно зберігати у таємниці.

Його не можна публікувати в GitHub або передавати стороннім особам.

У разі компрометації токен потрібно відкликати через BotFather та створити новий.

## Команди бота

Команди, створені через BotFather, лише відображаються в меню Telegram.

Їхня поведінка визначається програмою, яку ми напишемо.

## Адміністрування каналу

Для автоматичної публікації повідомлень бот повинен бути доданий адміністратором каналу.

Рекомендується надавати лише мінімально необхідні права.

Telegram Mini App

На поточному етапі виконано:

реєстрацію Mini App через BotFather;
прив'язку GitHub Pages до бота;
підключення Telegram WebApp API;
тестування роботи API у звичайному браузері.

Під час запуску у браузері WebApp API доступне, але службові дані Telegram (initData) відсутні.

Після запуску всередині Telegram очікується отримання службових даних користувача та середовища виконання.

Кожна гра Logic Games повинна містити:

README.md — опис гри
config.js — налаштування
tasks.js — набір завдань
script.js — логіка
style.css — оформлення
index.html — точка входу

assets/
images/
sounds/
themes/

Перед початком розробки кожної нової гри спочатку створюється сценарій (docs/Scenario.md), і лише після його затвердження починається програмування.

================ Доповнення до правил створення проэкту ==================================
одна механіка — одна папка;
дані відокремлені від логіки;
контент зберігається в content/set-XXX;
опубліковані набори не змінюються.
==========================================================================================

Стандартна схема кольорів Logic Games
:root {
--background-color: #F8F9FA; /_ Фон _/
--surface-color: #FFFFFF; /_ Картки, панелі _/
--primary-color: #2F80ED; /_ Основний колір _/
--border-color: #D9DDE3; /_ Межі клітинок _/
--text-color: #222222; /_ Основний текст _/

    --success-color:    #2E7D32;   /* Правильна відповідь */
    --error-color:      #C62828;   /* Помилка */

    --button-text:      #FFFFFF;

}
Правило 1. Якщо гра використовує набори даних, то вони зберігаються в каталозі content/ всередині папки гри.

Правило 2. Номер набору даних передається в полі game.set.

Правило 3. Якщо поле game.set відсутнє, гра не використовує набори даних.
